# 요소 분류
{: #element-classification}

요소 분류를 사용하면 운영 문서를 통해 빠르게 구문 분석하여 중요한 요소를 변환, 식별 및 분류할 수 있습니다. 최첨단 자연어 처리, 당사자(참조하는 사용자), 네이처(요소 유형) 및 카테고리(특정 클래스)가 문서의 요소에서 추출됩니다. 

요소 분류는 다음을 제공하도록 설계되었습니다. 

- 소프트웨어 조달 계약 및 규정 문서에 중점을 둔 계약의 자연어 이해
- 프로그램의 PDF를 어노테이션이 있는 JSON으로 변환하는 기능
- 주제별 전문 지식으로 정렬되는 법적 엔티티 및 카테고리의 식별

요소 분류는 섹션, 목록(번호 매기기 및 글머리표 지정), 각주 및 테이블을 식별하여 이러한 항목을 구조화된 HTML 형식으로 변환할 수 있는 프로그래밍 방식 PDF를 입력하기 위한 기능상으로 풍부한 통합 및 자동화된 Watson API 세트를 함께 제공합니다. 또한 이 구조화된 형식의 분류가 어노테이션되고 레이블 지정된 요소, 유형 및 카테고리를 사용하여 JSON으로 출력됩니다. 

요소 분류는 운용 중이든 휴지 중이든 암호화를 수행하여 데이터를 안전하게 전송합니다. IBM Cloud 보안에 대한 자세한 정보는 [IBM Cloud 서비스 설명 ![외부 링크 아이콘](../../icons/launch-glyph.svg "외부 링크 아이콘")](http://www.ibm.com/software/sla/sladb.nsf/searchsaas/?searchview&searchorder=4&searchmax=0&query=IBM+Bluemix+Service+Description){: new_window}을 참조하십시오.

## 분류 요구사항

요소 분류를 사용하여 문서를 분류하려면 구성 및 소스 문서가 다음 요구사항을 충족해야 합니다. 

- 분석할 파일은 PDF 형식입니다. 
- PDF 컨텐츠는 텍스트 양식으로 제공됩니다(스캔된 문서이지만 OCR이 사용되지 않아 구문 분석할 수 없음). 

  **참고:** 하나의 단어를 선택하기 위해 PDF 뷰어로 문서를 열고 **텍스트 선택** 도구를 사용하여 텍스트로 된 PDF를 식별할 수 있습니다. 문서에서 하나의 단어를 선택할 수 없는 경우 파일을 구문 분석할 수 없습니다. 

- 파일 크기는 50MB보다 크지 않습니다. 
- 보안 설정된 PDF(비밀번호를 사용하여 열기) 및 제한된 PDF 편집(비밀번호를 사용하여 편집)은 구문 분석될 수 없습니다.
- 사용자 정의 {{site.data.keyword.discoveryshort}} 구성은 `elements` 인리치먼트를 포함하여 작성되어야 합니다. 이 구성은 PDF 문서를 수집하는 경우에만 사용할 수 있습니다. 
- **라이트** 및 **표준** 플랜은 매월 최대 500페이지를 처리할 수 있습니다. 
- **프리미엄** 플랜에 등록된 서비스 인스턴스에는 사용할 수 없습니다. 

## 콜렉션 요구사항

요소 분류를 사용하려면 다음과 같이 특정 요구사항을 충족하도록 콜렉션을 구성해야 합니다. 

- 지정된 경우 `PDF` 변환 설정이 무시됩니다. 
- Microsoft Word 파일로 생략할 수 있는 `WORD` 변환 설정은 요소 분류를 지정할 때 수집될 수 없습니다. 
- 지정된 경우 `html` 정규화 설정이 무시됩니다. 
- `elements` 인리치먼트가 지정되면 문서 분할이 지원되지 않습니다. 
- {{site.data.keyword.discoveryshort}} 구성에서 `html` 필드는 `elements` 인리치먼트와 `contract`로 지정된 `model`로 강화되어야 합니다. 다음 예에서 `destination_field`는 `enriched_html`이지만 올바른 이름은 사용할 수 있습니다. 

```json
"enrichments": [
  {
    "source_field": "html",
    "destination_field": "enriched_html",
    "enrichment": "elements",
    "options": {
      "model": "contract"
    }
  }
]
```
{: codeblock}

이 옵션은 {{site.data.keyword.discoveryshort}} 도구를 사용하여 추가될 수 있으며 사용자 정의 구성을 작성하고 **요소 분류** 인리치먼트를 `html` 필드에 추가합니다. 

**참고:** 도구를 사용하여 **요소 분류**를 추가할 때 `destination_field`는 `enriched_html_elements`로 설정됩니다.

사용자 정의 구성이 작성된 후 모든 콜렉션에 사용될 수 있으며 사용자 정의 구성이 지정되는 한 문서를 업로드하는 모든 메소드를 사용할 수 있습니다. 콜렉션을 작성하고 문서를 업로드하는 데 익숙하지 않은 경우 [도구 시작하기](/docs/services/discovery/getting-started-tool.html)를 참조하십시오.

## 분류된 요소

문서가 요소 분류로 인덱싱되면 검색 가능한 문서의 일부로 `elements` 배열을 사용하여 리턴됩니다. 

`elements` 배열의 각 오브젝트는 요소 분류가 식별한 계약의 요소에 대해 설명합니다. 다음 코드는 일반 요소를 나타냅니다. 

```
{
   "sentence" : {
     "begin" : 34941,
     "end" : 35307
   },
   "sentence_text" : "A buyer must buy from the supplier.",
   "types" : [ {
     "label" : {
       "nature" : "Obligation",
       "party" : "Supplier"
     },
     "assurance" : "High"
   } ],
   "categories" : [ {
     "label : "Responsibilities",
     "assurance : "High"
     }
   ]
}
```

요소에는 네 개의 중요한 섹션이 있습니다. 

- `sentence_text` – 분석된 텍스트입니다. 
- `sentence` – 이 오브젝트는 변환된 HTML로 요소가 발견된 위치에 대해 설명하며, 여기에는 시작 문자 값과 종료 문자 값이 포함됩니다. 
- `types` – 이 배열은 요소에 대한 개념 및 영향을 주는 사용자에 대해 설명하며, 이는 하나 이상의 `party`(문장으로 영향을 받은 사용자) 및 `nature`(식별된 사용자에 대한 문장의 효과) 세트로 구성됩니다. 
- `categories` – 이 배열은 식별된 문장이 속하는 기능 카테고리를 나열합니다. 문장의 주제입니다. 

**참고**: 일부 문장은 유형 또는 카테고리에 속하지 않고, 이 경우 `types` 및 `categories` 배열은 비어 있는 상태로 리턴됩니다. 

**참고:** 일부 문장은 여러 주제를 다루므로 다중 `types` 및 `categories` 항목이 나열되며 리턴됩니다. 

또한 식별되는 당사자도 parties 배열로 정의됩니다. 

```
  "parties" : [ {
    "party" : "Customer",
    "role" : "Buyer"
  } ]
```

parties 배열의 각 요소에는 두 개의 중요한 섹션이 있습니다. 

- `party` – 문서 내의 당사자로 식별된 텍스트입니다. 
- `role` – 식별된 당사자의 역할입니다. 역할은 하위 도메인에 따라 변경됩니다. 가능한 역할 목록은 지정된 하위 도메인에 있는 정보를 참조하십시오. 특정 역할로 식별될 수 없는 당사자는 '알 수 없음'으로 레이블 지정됩니다. 


## 계약 요소 이해

요소 분류에서 구문 분석된 계약은 분석되어 식별된 각 요소로 리턴됩니다. 

다음 절에서는 리턴된 JSON이 분석을 설명하는 방법에 대해 설명합니다. 

### 유형

요소 `types` 정보는 오브젝트의 배열이며, 각 오브젝트에는 이 요소에 대해 한 쌍으로 식별된 네이처 및 당사자가 포함됩니다. 다음 표에서는 식별될 수 있는 가능한 `natures` 및 `parties`에 대해 설명합니다. 

네이처는 문장에 필요한 조치의 유형입니다. 

| **네이처** | **설명** |
| --- | --- |
| 정의 | 이 요소는 용어/관계/등에 대한 명확성을 추가합니다. 조치는 이 요소를 수행하는 데 필요하지 않고 당사자는 영향을 받지 않습니다. |
| 면책사항| 요소의 당사자는 요소의 특정 조항을 이행할 의무는 없지만 제외되지는 않습니다. |
| 제외 | 요소의 당사자는 요소에 있는 특정 조항을 이행하지 않습니다. |
| 의무 | 요소의 당사자는 요소의 조항을 이행해야 합니다. |
| 권한 | 요소의 당사자는 요소의 조항을 수신하도록 보장되지 않습니다. |

### 당사자

식별된 각 절의 경우 영향을 받은 당사자가 이름으로 식별됩니다. 식별된 각 당사자는 `role`로 분류됩니다. 당사자는 계약의 참여자입니다. 계약에 대해 식별될 수 있는 역할은 다음과 같습니다. 

| **역할** | **설명** |
| --- | --- |
| `Buyer` | 상품/서비스에 대한 비용을 지불해야 하는 당사자입니다. |
| `End User` | 실제 상품/서비스와 상호 작용하는 당사자로, Buyer와는 명백하게 구분됩니다. |
| `None` | 이 요소에 대해 식별되는 당사자가 없으며, 항상 정의 네이처와 쌍을 구성합니다. |
| `Supplier` | 상품/서비스를 제공해야 하는 당사자입니다. |

### 카테고리

카테고리는 문장의 주제를 정의합니다. 다음 카테고리가 식별될 수 있습니다. 

| **카테고리** | **설명** |
| --- | --- |
| `Amendments` | 조항을 변경하기 위해 조건을 규정한 원본 계약의 수정입니다. |
| `Asset Use` | 두 당사자가 사용하는 계약의 모든 자산에 대한 세부사항입니다. |
| `Assignments` | 당사자 간에 보유한 권한의 이동을 포함합니다. |
| `Audits` | 이 절을 통해 구매자가 제공된 서비스의 공급자를 조사하거나 점검할 수 있습니다. |
| `Communication` | 허용 가능한 통신 방법 및 연락처 정보에 대해 설명합니다. |
| `Confidentiality` | 누가 무엇을 어떻게 공유할 수 있는지와 같이 기밀이거나 개인 정보를 처리하는 방법에 대해 설명합니다. |
| `Business Continuity` | 인시던트의 경우 조직이 사전 정의된 레벨로 작업을 계속 전달하는 방법에 대해 설명합니다. |
| `Definitions` | 문서에 사용되는 용어를 정의하는 섹션입니다. |
| `Deliverables` | 작업의 끝에 전달될 항목 또는 서비스입니다. |
| `Delivery` | 프로젝트를 완료하기 위한 특정 스케줄 또는 프로세스입니다. |
| `Dispute Resolution` | 계약 당사자 간에 발생할 수 있는 분쟁과 해결 방법을 제공합니다. |
| `Force Majeure` | 분쟁 발생 시 두 당사자의 면책 조항입니다. |
| `Indemnification` | 조항이 위반되는 경우 해결 방법 또는 결과에 대해 설명합니다. |
| `Insurance` | 공급자가 이행해야 하는 보험 적용 범위의 레벨에 대해 설명합니다. |
| `Intellectual Property` | 특허, 저작권 및 상표에 관련된 조항입니다. 보다 일반적으로 발명, 저작권 또는 노하우와 관련될 수 있습니다. |
| `Liability` | 각 당사자의 책임에 대한 의무 및 제한사항에 대해 설명합니다. |
| `Miscellaneous` | 기타 카테고리에 맞지 않는 관심 분야에 대한 섹션입니다. |
| `Payment Terms & Billing` | 지불 금액 및 지불 스케줄에 대해 설명합니다. |
| `Pricing & Taxes` | 가격이 구성되고 세금이 적용되는 방법에 대해 설명합니다. |
| `Privacy` | 적용되는 개인정보 보호정책 규정에 대해 설명합니다. |
| `Responsibilities` | 각 당사자의 책임에 대해 설명합니다. |
| `Safety and Security` | 당사자의 피해를 방지하는 방법에 대해 설명합니다. 인적 및 물리적 자산 안전이 모두 포함됩니다. |
| `Scope of Work` | 작업 명세에 자세하게 설명된 당사자가 수행할 항목에 대해 설명합니다. |
| `Subcontracts` | 요구사항을 수행하기 위해 연관된 써드파티와 관련됩니다. |
| `Term & Termination` | 어떤 일이 발생하는 시간 및 종료될 수 있는 조건입니다. |
| `Warranties` | 공급자가 제품 작동 방식을 보증합니다. |

### 보증

요소 분류로 식별된 모든 항목(유형 또는 카테고리)에는 `assurance` 등급이 지정됩니다. 가능한 보증 값에 대한 설명은 다음과 같습니다. 

| **보증** | **설명** |
| --- | --- |
| `High` | 지정된 분류가 컨텐츠를 나타내는 중요한 근거가 있습니다. |
| `Low` | 분류를 지원하는 일부 근거가 있으나 확인을 위해 추가 검토가 필요할 수 있습니다. |
